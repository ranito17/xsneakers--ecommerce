# Bug Fix Workflow

You are fixing a bug in a production fullstack e-commerce app (React + Node.js + MySQL).

## Steps (follow in order)

1. **Identify** — Read the relevant file(s). Find the exact root cause. Do not assume.
2. **Explain** — In 2–3 sentences, describe what is broken and why.
3. **Confirm** — If the cause is unclear or multiple fixes are possible, ASK the user before proceeding.
4. **Fix** — Change ONLY the root cause. Do not touch unrelated code.
5. **Show** — Output the FULL updated file. At the top, list exactly what lines changed.

## Constraints
- Do not refactor anything beyond the fix
- Do not rename variables or functions
- Do not add new dependencies unless the fix requires it
- Do not touch business logic unless the bug is in the logic itself
- PayPal must remain in SANDBOX mode
- Preserve all existing imports, routes, and component structure
