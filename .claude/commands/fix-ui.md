# UI Fix Workflow

You are fixing a UI/design issue in a production React e-commerce frontend.

## Steps (follow in order)

1. **Scope** — Identify exactly which element, component, or style is affected.
2. **Explain** — In 1–2 sentences, describe what you will change and nothing more.
3. **Fix** — Modify ONLY the requested visual/layout change.
4. **Show** — Output the FULL updated component. List the CSS or JSX lines changed at the top.

## Constraints
- Do NOT redesign the whole component
- Do NOT touch logic, state, handlers, or API calls
- Do NOT change component structure, props, or imports unless the fix requires it
- Do NOT add new dependencies
- Keep changes surgical — one element at a time
- If the requested change could break responsiveness or another UI state (hover, mobile, dark mode), mention it before applying
