================================================================================
תיעוד מקיף של מערכת ה-Backend - מערכת ניהול חנות נעליים
================================================================================

תוכן עניינים:
1. סקירה כללית
2. מבנה הקבצים
3. מודלים (Models)
4. בקרים (Controllers)
5. נתיבים (Routes) וחיבורים ל-Frontend
6. שירותים (Services)
7. Middleware
8. מסד נתונים

================================================================================
1. סקירה כללית
================================================================================

המערכת בנויה על Node.js עם Express, ומשתמשת ב-MySQL כמסד נתונים.
המערכת מספקת API מלא לניהול חנות נעליים, כולל:
- ניהול משתמשים והרשאות
- ניהול מוצרים וקטגוריות
- ניהול הזמנות ותשלומים
- ניהול עגלת קניות
- מערכת הודעות
- אנליטיקס ודוחות
- מעקב פעילות מנהלים

================================================================================
2. מבנה הקבצים
================================================================================

backend/src/
├── app.js                    - קובץ האפליקציה הראשי, מגדיר את השרת והנתיבים
├── config/                   - קבצי הגדרות
│   ├── config.js            - הגדרות כלליות (פורט, JWT, CORS, וכו')
│   └── database.js          - חיבור למסד הנתונים
├── controllers/              - בקרים - לוגיקת עסקים וטיפול בבקשות
├── models/                   - מודלים - פעולות מסד נתונים
├── routes/                   - נתיבים - הגדרת API endpoints
├── services/                 - שירותים - לוגיקה חיצונית (אימייל, PayPal)
├── middleware/               - Middleware - אימות, לוגים, שגיאות
└── uploads/                  - תיקיית העלאת קבצים

================================================================================
3. מודלים (Models)
================================================================================

כל מודל מנהל את הפעולות הקשורות לטבלה מסוימת במסד הנתונים.

3.1 User.js - מודל המשתמשים
----------------------------
תפקיד: ניהול כל הפעולות הקשורות למשתמשים במסד הנתונים

פונקציות עיקריות:
- findUser(email, password) - מציאת משתמש ואימות סיסמה
  * שאילתה: SELECT password FROM users WHERE email = ?
  * בודקת סיסמה עם bcrypt
  * מחזירה: אובייקט משתמש או null
  
- findUserByEmail(email) - מציאת משתמש לפי אימייל
  * שאילתה: SELECT * FROM users WHERE email = ?
  * מחזירה: אובייקט משתמש או null
  
- createUser(userData) - יצירת משתמש חדש
  * שאילתה: INSERT INTO users (full_name, email, address, phone_number, password, role)
  * מצפינה סיסמה עם bcrypt
  * משתמשת בטרנזקציה
  * מחזירה: userId (insertId)
  
- createPasswordResetToken(email) - יצירת טוקן לאיפוס סיסמה
  * יוצרת טוקן אקראי עם crypto
  * מעדכנת את המשתמש עם הטוקן ותאריך תפוגה (שעה אחת)
  * שאילתה: UPDATE users SET reset_token = ?, reset_token_expiry = ? WHERE email = ?
  
- findUserByResetToken(resetToken) - מציאת משתמש לפי טוקן
  * שאילתה: SELECT * FROM users WHERE reset_token = ? AND reset_token_expiry > NOW()
  * מחזירה: אובייקט משתמש או null
  
- resetPassword(resetToken, newPassword) - איפוס סיסמה
  * בודקת שהטוקן תקין
  * מצפינה סיסמה חדשה
  * מעדכנת סיסמה ומנקה טוקן
  * שאילתה: UPDATE users SET password = ?, reset_token = NULL, reset_token_expiry = NULL WHERE id = ?
  
- getAllUsers() - קבלת כל המשתמשים
  * שאילתה: SELECT id, full_name, email, address, phone_number, role, created_at FROM users
  * מחזירה: מערך של משתמשים (ללא סיסמאות)
  
- deleteUser(userId) - מחיקת משתמש
  * בודקת אם למשתמש יש הזמנות (לא מאפשרת מחיקה אם יש)
  * שאילתה: DELETE FROM users WHERE id = ?
  * מחזירה: true אם הצליחה
  
- updateProfile(userId, profileData) - עדכון פרטי משתמש
  * מעדכנת שם וטלפון
  * שאילתה דינמית: UPDATE users SET full_name = ?, phone_number = ? WHERE id = ?
  
- updateAddress(userId, addressData) - עדכון כתובת
  * שומרת כתובת בפורמט JSON
  * שאילתה: UPDATE users SET address = ? WHERE id = ?
  
- changePassword(userId, currentPassword, newPassword) - שינוי סיסמה
  * מאמתת סיסמה נוכחית
  * מצפינה סיסמה חדשה
  * שאילתה: UPDATE users SET password = ? WHERE id = ?
  
- getWishlist(userId) - קבלת רשימת משאלות
  * שאילתה: SELECT wishlist FROM users WHERE id = ?
  * מחזירה: מערך של product IDs
  
- addToWishlist(userId, productId) - הוספת מוצר לרשימת משאלות
  * בודקת אם המוצר כבר קיים
  * מעדכנת את רשימת המשאלות ב-JSON
  * שאילתה: UPDATE users SET wishlist = ? WHERE id = ?
  
- removeFromWishlist(userId, productId) - הסרת מוצר מרשימת משאלות
  * מסננת את המוצר מהרשימה
  * מעדכנת את רשימת המשאלות
  * שאילתה: UPDATE users SET wishlist = ? WHERE id = ?

3.2 Product.js - מודל המוצרים
-----------------------------
תפקיד: ניהול כל הפעולות הקשורות למוצרים במסד הנתונים

פונקציות עיקריות:
- getAllProducts() - קבלת כל המוצרים
  * שאילתה מורכבת עם JOIN לקטגוריות וסטטיסטיקות מכירות
  * מחשבת: total_quantity_sold, total_revenue, total_sales
  * מעבדת תמונות וגדלים (JSON)
  * מחזירה: מערך מוצרים מעובדים עם תמונות מלאות וגדלים
  
- getProductById(id) - קבלת מוצר לפי מזהה
  * שאילתה: SELECT p.*, c.category_name FROM products p LEFT JOIN categories c
  * מעבדת תמונות וגדלים
  * מחזירה: אובייקט מוצר מעובד
  
- createProduct(productData) - יצירת מוצר חדש
  * מחשבת total_stock מגדלים
  * שומרת גדלים כ-JSON
  * שאילתה: INSERT INTO products (category_id, name, price, stock_quantity, description, sizes, image_urls, is_active)
  * מחזירה: result object עם insertId
  
- updateProduct(id, productData) - עדכון מוצר
  * מעדכנת את כל השדות כולל גדלים
  * שאילתה: UPDATE products SET category_id = ?, name = ?, price = ?, ...
  
- deleteProduct(id) - מחיקת מוצר
  * בודקת אם המוצר פעיל (לא מאפשרת מחיקה אם פעיל)
  * שאילתה: DELETE FROM products WHERE id = ?
  
- getLowStockProducts(threshold) - מוצרים עם מלאי נמוך (סה"כ)
  * שאילתה: SELECT * FROM products WHERE stock_quantity <= ?
  * מחזירה: מוצרים עם מלאי נמוך
  
- getProductsWithLowStockSizes(sizeThreshold) - מוצרים עם גדלים במלאי נמוך
  * שאילתה מורכבת: SELECT * FROM products ORDER BY name
  * מעבדת כל מוצר ובודקת גדלים
  * מחזירה רק מוצרים שיש להם לפחות גדל אחד במלאי נמוך
  * כוללת תמונות מעובדות
  
- updateSizeQuantityForProduct(productId, size, newQuantity) - עדכון כמות גדל
  * מעדכנת כמות גדל ספציפי
  * מחשבת מחדש total_stock
  * שאילתה: UPDATE products SET sizes = ?, stock_quantity = ? WHERE id = ?
  
- addSizeToProduct(productId, size, quantity) - הוספת גדל למוצר
  * מוסיפה גדל חדש או מעדכנת קיים
  * מחשבת מחדש total_stock
  
- decreaseStockForSize(productId, size, quantityToDecrease) - הפחתת מלאי לאחר הזמנה
  * מפחיתה כמות מגדל ספציפי
  * בודקת אם יש מספיק מלאי
  * מעדכנת total_stock
  
- getStockForSize(productId, size) - קבלת מלאי לגדל ספציפי
  * מחזירה: כמות זמינה לגדל
  
- deactivateProduct(productId) - השבתת מוצר
  * שאילתה: UPDATE products SET is_active = 0 WHERE id = ?
  
- activateProduct(productId) - הפעלת מוצר
  * שאילתה: UPDATE products SET is_active = 1 WHERE id = ?

3.3 Order.js - מודל ההזמנות
-----------------------------
תפקיד: ניהול כל הפעולות הקשורות להזמנות במסד הנתונים

פונקציות עיקריות:
- placeOrder(orderData) - יצירת הזמנה חדשה
  * בודקת מלאי לפני יצירת הזמנה
  * יוצרת מספר הזמנה בפורמט ORD-YYYY-XXX
  * מחשבת תאריך אספקה משוער (חודש מהיום)
  * שומרת כתובת כ-JSON
  * שאילתות:
    - INSERT INTO orders (user_id, total_amount, payment_status, order_number, status, arrival_date_estimated, address)
    - INSERT INTO order_items (order_id, product_id, quantity, selected_size)
  * מפחיתה מלאי מכל מוצר
  * מחזירה: { success: true, orderId, orderNumber }
  
- getAllOrders(sizeFilter) - קבלת כל ההזמנות
  * שאילתה מורכבת עם JOIN למשתמשים
  * תמיכה בסינון לפי גדל
  * מעבדת כתובות JSON
  * מחזירה: מערך הזמנות
  
- getOrderById(orderId) - קבלת הזמנה לפי מזהה
  * שאילתה: SELECT o.*, u.full_name, u.email FROM orders o JOIN users u
  * מעבדת כתובת JSON
  * מחזירה: אובייקט הזמנה
  
- getUserOrderById(orderId, userId) - קבלת הזמנה של משתמש ספציפי
  * בודקת שההזמנה שייכת למשתמש
  * שאילתה: SELECT * FROM orders WHERE order_id = ? AND user_id = ?
  
- updateOrder(orderId, orderData) - עדכון הזמנה
  * מעדכנת סכום, סטטוס, ומספר מעקב
  * שאילתה: UPDATE orders SET total_amount = ?, status = ?, tracking_number = ?
  
- updateOrderStatus(orderId, status) - עדכון סטטוס הזמנה
  * שאילתה: UPDATE orders SET status = ? WHERE order_id = ?
  
- getUserOrders(userId) - קבלת הזמנות של משתמש
  * שאילתה: SELECT * FROM orders WHERE user_id = ? ORDER BY created_at DESC
  
- getUserOrderCount(userId) - ספירת הזמנות של משתמש
  * שאילתה: SELECT COUNT(*) FROM orders WHERE user_id = ?
  
- getOrderItems(orderId) - קבלת פריטי הזמנה
  * שאילתה מורכבת עם JOIN למוצרים
  * מעבדת תמונות וכתובות
  * מחזירה: מערך פריטי הזמנה
  
- getDashboardStats() - סטטיסטיקות לדשבורד
  * שאילתות מורכבות:
    - סטטיסטיקות הזמנות (סה"כ, הכנסות, לפי סטטוס)
    - הזמנות אחרונות (8)
    - מוצרים מובילים (8)
    - הכנסות חודשיות (6 חודשים אחרונים)
  * מחזירה: אובייקט עם כל הסטטיסטיקות
  
- getBestSellers(limit) - מוצרים מובילים
  * שאילתה מורכבת המחשבת מכירות לפי טווח זמן מהגדרות
  * מדרגת לפי כמות נמכרה ואז הכנסות
  * מחזירה: { products, timeRange, limit, totalFound }
  
- getRevenueAnalytics(startDate, endDate, groupBy) - אנליטיקס הכנסות
  * מקבצת הכנסות לפי תקופה (יום/שבוע/חודש/שנה)
  * שאילתה: SELECT DATE_FORMAT(created_at, ?) as period, SUM(total_amount) as revenue
  * מחזירה: מערך נתונים לפי תקופה
  
- getProductAnalytics(startDate, endDate) - אנליטיקס מוצרים
  * שאילתות מורכבות:
    - ביצועי מוצרים (מכירות, הכנסות, מלאי נותר)
    - ביצועי קטגוריות
  * מחזירה: { productPerformance, categoryPerformance }
  
- getUserAnalytics(startDate, endDate) - אנליטיקס משתמשים
  * מחשב: משתמשים חדשים, משתמשים פעילים, משתמשים חוזרים
  * שאילתות: COUNT משתמשים לפי קריטריונים
  * מחזירה: { newUsers, activeUsers, returningUsers, growthRate, userGrowth }

3.5 Cart.js - מודל עגלת הקניות
--------------------------------
תפקיד: ניהול עגלת הקניות של המשתמשים

פונקציות עיקריות:
- getUserCart(userId) - קבלת עגלת קניות של משתמש
  * יוצרת עגלה אם לא קיימת
  * שאילתה: SELECT * FROM carts WHERE user_id = ?
  * שאילתה: SELECT ci.*, p.* FROM cart_items ci JOIN products p
  * מחזירה: { cartId, items, totalCost }
  
- addToCart(userId, productId, quantity, selectedSize) - הוספת פריט לעגלה
  * בודקת אם פריט כבר קיים (עם אותו גדל)
  * מעדכנת כמות או מוסיפה פריט חדש
  * מעדכנת סה"כ עגלה
  * שאילתות:
    - INSERT INTO cart_items (cart_id, product_id, quantity, price, selected_size)
    - UPDATE cart_items SET quantity = ?
  
- updateQuantity(userId, cartItemId, quantity) - עדכון כמות פריט
  * בודקת שהפריט שייך למשתמש
  * מעדכנת כמות ומחשב מחדש סה"כ
  * שאילתה: UPDATE cart_items SET quantity = ? WHERE cart_item_id = ?
  
- removeFromCart(userId, cartItemId) - הסרת פריט מעגלה
  * בודקת שהפריט שייך למשתמש
  * מוחקת פריט ומעדכנת סה"כ
  * שאילתה: DELETE FROM cart_items WHERE cart_item_id = ?
  
- clearCart(userId) - ניקוי עגלה
  * מוחקת כל הפריטים מעגלה
  * שאילתה: DELETE FROM cart_items WHERE cart_id = ?

3.6 Settings.js - מודל ההגדרות
--------------------------------
תפקיד: ניהול הגדרות המערכת (key-value)

פונקציות עיקריות:
- getSettings() - קבלת כל ההגדרות
  * שאילתה: SELECT setting_key, setting_value FROM settings
  * ממירה ערכים לטיפוסים (מספר, בוליאני, JSON)

- getSetting(key, defaultValue) - קבלת הגדרה אחת
  * שאילתה: SELECT setting_value FROM settings WHERE setting_key = ?
  * מחזירה: ערך ההגדרה או default
  
- updateSettings(settingsData) - עדכון הגדרות
  * משתמשת ב-INSERT ... ON DUPLICATE KEY UPDATE
  * שאילתה: INSERT INTO settings (setting_key, setting_value) VALUES (?, ?) ON DUPLICATE KEY UPDATE
  
- getSettingsWithDescriptions() - קבלת הגדרות עם תיאורים
  * מחזירה: אובייקט עם value ו-description לכל הגדרה

3.7 Message.js - מודל ההודעות
-------------------------------
תפקיד: ניהול הודעות קשר (לקוחות/אורחים/אדמינים)

פונקציות עיקריות:
- getAllMessages() - קבלת כל ההודעות
  * שאילתה מורכבת עם JOIN להזמנות, מוצרים, משתמשים
  * מחזירה: מערך הודעות
  
- getMessageById(messageId) - קבלת הודעה לפי מזהה
  * שאילתה: SELECT cm.*, o.order_number, p.name, u.full_name FROM contact_messages cm
  
- createMessage(messageData) - יצירת הודעה חדשה
  * שאילתה: INSERT INTO contact_messages (message_type, sender_user_id, sender_name, ...)
  * מחזירה: { messageId, ...messageData }
  
- updateMessageStatus(messageId, status) - עדכון סטטוס הודעה
  * שאילתה: UPDATE contact_messages SET status = ? WHERE message_id = ?
  
- deleteMessage(messageId) - מחיקת הודעה
  * בודקת שההודעה resolved או archived
  * שאילתה: DELETE FROM contact_messages WHERE message_id = ?
  
- getUnreadCount() - ספירת הודעות שלא נקראו
  * שאילתה: SELECT COUNT(*) FROM contact_messages WHERE status = 'new'

3.8 Activity.js - מודל מעקב פעילות
------------------------------------
תפקיד: מעקב אחר פעולות מנהלים במערכת

פונקציות עיקריות:
- create(activityData) - יצירת רשומת פעילות
  * שאילתה: INSERT INTO activities (user_id, action_type, entity_type, entity_id, description)
  
- getAll(filters) - קבלת פעילויות עם סינון
  * שאילתה מורכבת עם JOIN למשתמשים
  * תמיכה בסינון לפי: action_type, entity_type, user_id, user_email, תאריכים
  * תמיכה ב-pagination (limit, offset)
  * מחזירה: מערך פעילויות
  
- getCount(filters) - ספירת פעילויות
  * שאילתה: SELECT COUNT(*) FROM activities WHERE ...
  
- getStatistics(days_back) - סטטיסטיקות פעילות
  * שאילתה: SELECT action_type, entity_type, COUNT(*) FROM activities GROUP BY ...
  
- deleteOldActivities(daysOld) - מחיקת פעילויות ישנות
  * שאילתה: DELETE FROM activities WHERE created_at < DATE_SUB(NOW(), INTERVAL ? DAY)

================================================================================
4. בקרים (Controllers)
================================================================================

כל בקר מקבל בקשות HTTP, מטפל בהן, קורא למודל המתאים, ומחזיר תגובה.

4.1 userController.js
---------------------

================================================================================
7. Middleware
================================================================================

7.1 auth.js
-----------
תפקיד: אימות משתמשים באמצעות JWT

- בודקת token ב-cookie
- מפענחת JWT
- מוסיפה req.user עם פרטי המשתמש
- מחזירה 401 אם לא מאומת

7.2 adminAuth.js
----------------
תפקיד: בדיקת הרשאות אדמין

- בודקת ש-req.user.role === 'admin'
- מחזירה 403 אם לא אדמין

7.3 activityLogger.js
---------------------
תפקיד: רישום פעולות מנהלים

- trackActivity(options) - middleware לרישום פעילות
- משמש ב-routes של פעולות אדמין
- יוצר רשומות ב-activities table

7.4 error.js
------------
תפקיד: טיפול בשגיאות

- לוכד שגיאות מכל הבקשות
- מחזיר תגובות שגיאה אחידות

7.5 logger.js
-------------
תפקיד: רישום בקשות

- רושם כל בקשה לשרת
- כולל: method, path, status, time

7.6 upload.js
-------------
תפקיד: ניהול העלאת קבצים

- productUpload - multer למוצרים
- categoryUpload - multer לקטגוריות
- שמירת קבצים ב-uploads/

================================================================================
8. מסד נתונים
================================================================================

טבלאות עיקריות:
- users - משתמשים
- products - מוצרים
- categories - קטגוריות
- orders - הזמנות
- order_items - פריטי הזמנה
- carts - עגלות קניות
- cart_items - פריטי עגלה
- contact_messages - הודעות קשר
- settings - הגדרות (key-value)
- activities - פעילויות מנהלים

קשרים:
- products.category_id -> categories.category_id
- orders.user_id -> users.id
- order_items.order_id -> orders.order_id
- order_items.product_id -> products.id
- cart_items.cart_id -> carts.cart_id
- cart_items.product_id -> products.id
- carts.user_id -> users.id
- contact_messages.sender_user_id -> users.id (nullable)
- contact_messages.order_id -> orders.order_id (nullable)
- contact_messages.product_id -> products.id (nullable)
- activities.user_id -> users.id (nullable)

סיכום
================================================================================

המערכת מספקת API מלא לניהול חנות נעליים עם:
- ניהול משתמשים מלא (הרשמה, התחברות, פרופיל, רשימת משאלות)
- ניהול מוצרים וקטגוריות (CRUD מלא)
- ניהול הזמנות (יצירה, עדכון סטטוס, מעקב)
- עגלת קניות
- מערכת הודעות
- אנליטיקס ודוחות
- מעקב פעילות מנהלים
- תשלומי PayPal
- הגדרות מערכת

כל הנתיבים מוגנים בהתאם (ציבורי, מאומת, אדמין) ומחוברים לרכיבי Frontend המתאימים.
